reduce Operator (Examples)

2020년 10월 6일 오후 6:18

These examples use the following predefined variables:

  • add-price: [get[price]multiply{!!quantity}add<accumulator>]
  • num-items: [get[quantity]add<accumulator>]
  • join-with-commas: [<index>compare:number:gt[0]then<accumulator>addsuffix[, ]addsuffix<currentTiddler>else<currentTiddler>]

They also use the following data tiddlers:

  • Brownies quantity: 1, price: 4.99
  • Chick Peas quantity: 5, price: 1.32
  • Milk quantity: 12, price: 0.46
  • Rice Pudding quantity: 4, price: 2.66

Number of items:

[tag[shopping]reduce<num-items>]

Total price:

[tag[shopping]reduce<add-price>]

Using <index> to act differently on the first item than the rest:

[tag[shopping]reduce<join-with-commas>]

Empty input, no suffix:

[tag[non-existent]reduce<add-price>]

Note how the output contains a single item with no text. This is not "empty output" for the purposes of the else operator.

Remember that reduce always produces output, so else will never trigger after reduce.

Empty input, no suffix, followed by else:

[tag[non-existent]reduce<add-price>else[0]]

Note how the output still contains a single item with no text: else did not trigger. If you want the value to be 0 when reduce has no items to process, you need to specify 0 as the initial value by passing it as a suffix to reduce.

Empty input, suffix provided:

[tag[non-existent]reduce:0<add-price>]
